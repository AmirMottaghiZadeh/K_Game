# کارآموزی عرصه

یک بازی آموزشی تحت وب برای تمرین اطلاعات دارویی در دوره کارآموزی عرصه داروسازی. پروژه به صورت Static Web App نوشته شده و می‌تواند هم به شکل کاملا local داخل مرورگر کار کند، هم با اتصال به Supabase حساب کاربری global، ذخیره‌سازی ابری و لیگ رقابتی بین کاربران داشته باشد.

## لینک اجرای آنلاین

بعد از فعال‌سازی GitHub Pages با GitHub Actions، پروژه از این آدرس در دسترس خواهد بود:

https://amirmottaghizadeh.github.io/K_Game/

## قابلیت‌های اصلی

- چهار مدل کوییز دارویی:
  - با غذا / بی غذا
  - نام تجاری / ژنریک
  - اندیکاسیون
  - عوارض جانبی
- حالت بازی تصادفی با انتخاب تعداد سؤال
- حالت بازی با تمام سؤال‌های یک موضوع
- تایمر قابل تنظیم برای هر سؤال
- مقدار پیش‌فرض تایمر: 30 ثانیه
- پنل تنظیمات با انتخاب تم روشن و تاریک
- داشبورد آماری برای امتیازها، دقت و خطاها
- ذخیره خطاها برای مرور دوباره
- خروجی گرفتن از گزارش فعالیت‌ها به صورت JSON
- حساب کاربری با email/password و username
- ذخیره فعالیت‌ها، امتیازها، خطاها و نتایج لیگ برای هر کاربر
- لیگ رقابتی با 50 سؤال تصادفی برای هر موضوع
- رتبه‌بندی لیگ بر اساس امتیاز و زمان باقی‌مانده
- پشتیبانی از نصب به شکل PWA با manifest و service worker

## مدل امتیازدهی لیگ

در لیگ، برای هر بار مسابقه 50 سؤال تصادفی از موضوع انتخاب‌شده ساخته می‌شود.

امتیاز نهایی لیگ از دو بخش تشکیل می‌شود:

- `scorePerQuestion`: امتیاز خام کاربر تقسیم بر 50
- `timeBonus`: مجموع زمان باقی‌مانده سؤال‌های درست تقسیم بر 50

برای رتبه‌بندی، مقدار `leagueRating` استفاده می‌شود:

```text
leagueRating = scorePerQuestion + timeBonus
```

در جدول لیگ، بهترین نتیجه هر کاربر نمایش داده می‌شود. اگر دو کاربر رتبه برابر داشته باشند، امتیاز اصلی بالاتر و سپس نتیجه جدیدتر اولویت می‌گیرد.

## معماری پروژه

این پروژه بدون build step و بدون فریم‌ورک اجرا می‌شود. فایل `index.html` مستقیما در مرورگر باز می‌شود و تمام منطق برنامه در JavaScript سمت کلاینت قرار دارد.

فایل‌های مهم:

- `index.html`: ساختار صفحه، فرم‌ها، تب‌ها و بارگذاری scriptها
- `styles.css`: طراحی رابط کاربری، تم روشن، تم تاریک و layout responsive
- `app.js`: منطق بازی، حساب کاربری، تایمر، لیگ، ذخیره‌سازی و اتصال Supabase
- `drugs-data.js`: داده‌های مربوط به زمان مصرف دارو نسبت به غذا
- `drug-topics-data.js`: داده‌های موضوعی برای نام ژنریک، اندیکاسیون و عوارض
- `manifest.webmanifest`: تنظیمات PWA
- `service-worker.js`: cache فایل‌های اصلی برای اجرای سریع‌تر و حالت PWA
- `supabase-config.js`: تنظیمات اتصال به Supabase
- `supabase-schema.sql`: اسکیما، policyها و triggerهای لازم برای Supabase
- `assets/`: آیکن‌ها و فایل‌های تصویری پروژه

## حالت local و حالت global

اگر `supabase-config.js` خالی باشد، برنامه در حالت local اجرا می‌شود. در این حالت حساب‌ها و فعالیت‌ها فقط در همان مرورگر و همان دستگاه ذخیره می‌شوند.

اگر `supabase-config.js` با اطلاعات Supabase پر شود، حالت global فعال می‌شود. در این حالت کاربر با ایمیل و پسورد وارد می‌شود و داده‌هایش از دستگاه‌های مختلف قابل دسترسی خواهد بود.

نمونه فایل تنظیمات:

```js
window.KARAMOZI_SUPABASE = {
  url: "https://YOUR_PROJECT_ID.supabase.co",
  anonKey: "YOUR_SUPABASE_ANON_PUBLIC_KEY",
};
```

## راه‌اندازی Supabase

1. وارد Supabase شوید و یک پروژه جدید بسازید.
2. از قسمت SQL Editor، محتوای فایل `supabase-schema.sql` را اجرا کنید.
3. از Project Settings، مقدارهای Project URL و anon public key را بردارید.
4. این دو مقدار را داخل `supabase-config.js` قرار دهید.
5. در Authentication > URL Configuration، مقدار Site URL را روی لینک GitHub Pages بگذارید:

```text
https://amirmottaghizadeh.github.io/K_Game/
```

6. اگر Email Confirmation را روشن گذاشته‌اید، کاربر بعد از ثبت‌نام باید ایمیل خود را تایید کند و سپس وارد شود.
7. اگر نمی‌خواهید تایید ایمیل لازم باشد، در تنظیمات Auth آن را غیرفعال کنید.

## جدول‌های Supabase

فایل `supabase-schema.sql` این جدول‌ها را می‌سازد:

- `profiles`: اطلاعات عمومی کاربر شامل username و email
- `user_states`: وضعیت ذخیره‌شده هر کاربر شامل امتیازها، خطاها و فعالیت‌ها
- `league_results`: نتایج هر مسابقه لیگ برای ساخت جدول رقابتی

روی جدول‌ها Row Level Security فعال است. کاربر فقط می‌تواند state خودش را بخواند و تغییر دهد، اما profileها و نتایج لیگ برای نمایش leaderboard قابل خواندن هستند.

## اجرای local

چون پروژه static است، می‌توانید مستقیم `index.html` را در مرورگر باز کنید. برای اجرای با سرور local هم می‌توانید از یک static server ساده استفاده کنید:

```bash
python3 -m http.server 8080
```

سپس آدرس زیر را باز کنید:

```text
http://127.0.0.1:8080/
```

## دیپلوی روی GitHub Pages

برای انتشار روی GitHub Pages:

1. پروژه را روی GitHub push کنید.
2. در تنظیمات repository وارد Pages شوید.
3. Source را روی GitHub Actions یا branch مناسب تنظیم کنید.
4. بعد از اجرای موفق workflow، آدرس نهایی پروژه این خواهد بود:

```text
https://amirmottaghizadeh.github.io/K_Game/
```

اگر فایل `supabase-config.js` را بعد از deploy تغییر دادید، بهتر است نسخه cache در `service-worker.js` را هم افزایش دهید تا مرورگر فایل جدید را سریع‌تر دریافت کند.

## نکات امنیتی

- مقدار `anonKey` در Supabase برای استفاده عمومی در frontend طراحی شده است و قرار گرفتن آن در GitHub Pages طبیعی است.
- هیچ service role key یا کلید محرمانه‌ای نباید داخل پروژه قرار بگیرد.
- پسورد کاربران در Supabase Auth مدیریت می‌شود و داخل فایل‌های پروژه ذخیره نمی‌شود.
- در حالت local، حساب‌ها فقط برای استفاده شخصی و مرورگری مناسب هستند و global محسوب نمی‌شوند.

## شخصی‌سازی داده‌ها

برای اضافه یا اصلاح کردن سؤال‌ها، داده‌ها را در این فایل‌ها تغییر دهید:

- `drugs-data.js`
- `drug-topics-data.js`

بعد از تغییر داده‌ها، اگر پروژه روی GitHub Pages منتشر شده باشد، فایل‌ها را commit و push کنید تا نسخه جدید deploy شود.

## وضعیت فعلی پروژه

پروژه آماده اجرا به شکل static است. برای استفاده کامل از حساب‌های global و leaderboard واقعی، فقط باید Supabase project ساخته شود، SQL schema اجرا شود و مقدارهای `supabase-config.js` تکمیل شوند.
